# MOV 游戏设计文档 v0.1

## 核心理念

**"无论你发出什么声音，都能转化为一个技能"**

核心体验是「发现」而非「记忆」。玩家像在学一门新语言，通过游戏反馈逐步归纳出声音与魔法的对应规律。

参考游戏：In Verbis Virtus（Steam）

---

## 战斗系统

- 实时战斗 + 有限次数时停施法（混合制）
- 时停消耗独立资源「专注值（Focus）」
- 时停期间玩家可以不受压力地发出咒语音节

## 咒语系统

### 结构

音节序列组合，1~3个音节均有效：

| 音节数 | 效果 |
|---|---|
| 1个音节 | 基础技能，弱但释放快 |
| 2个音节 | 标准技能 |
| 3个音节 | 强化技能，强但需要更长时间 |

### 槽位定义

| 槽位 | 类型 | 说明 |
|---|---|---|
| 第一音节（必须）| 元素 | 决定技能的「质」 |
| 第二音节（可选）| 形态 | 决定技能「怎么打」 |
| 第三音节（可选）| 修饰 | 决定技能的特殊效果 |

### 元素（4系）

| 元素 | 定位 |
|---|---|
| 火 | 高爆发，单体/小范围 |
| 冰 | 控制为主，低伤害 |
| 雷 | 范围/连锁，中等伤害 |
| 土 | 防御/回复，低机动 |

元素之间暂无相克关系，各自独立。

### 形态（6种）

| 形态 | 效果 |
|---|---|
| 弹 | 单体弹射，直线飞行 |
| 爆 | 以目标为中心范围爆炸 |
| 盾 | 护住自身，吸收伤害/附加效果 |
| 刃 | 近战挥砍，扇形范围 |
| 域 | 在地面布置持续区域 |
| 愈 | 对自身施加回复/增益 |

### 修饰（6种）

| 修饰 | 效果 |
|---|---|
| 强 | 伤害/效果×1.5，消耗更多专注值 |
| 连 | 连续释放2~3次 |
| 穿 | 弹射物穿透敌人 |
| 迟 | 延迟爆发，可布置陷阱 |
| 散 | 扩散为多个小型版本 |
| 追 | 弹射物追踪最近敌人 |

### 映射揭示机制（A2）

- 初期隐藏规则，玩家乱喊时系统给出反馈（如「你触发了火属性」）
- 随游戏进程逐渐揭示规律
- 游戏内有「咒语图鉴」，已发现的音节-属性关系会记录其中

## 敌人系统

### 三层设计

1. **形态弱点**：每个敌人有特定形态弱点（而非元素），引导玩家思考「用什么形态」
2. **状态效果**：
   - 冰系 → 减速 / 冻结
   - 火系 → 燃烧（持续伤害）
   - 雷系 → 麻痹
   - 土系 → 击退
3. **行为前摇**：敌人有明显的行动前摇动画，给玩家打断/反制的时机
